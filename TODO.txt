- Fix function scope deriving from caller scope instead of definition scope.
- Fix expressions that resolve to "NONE" not returning a variant of type "NONE" but instead returning an empty sequence.

- Add ability to define arguments in functions which can be used like any other variable.
- Resolve references with `.` characters preceeding into string `MAP` accessors. Using `.` is a more convenient way to access object members (no string literal accessor needed).
	For example, currently `IO:'print':['Hello World']` is the only member access syntax.
	But `IO.print:['Hello World']` should be valid too.


- Add standard library which can be compiled to pick & choose what you will actually use. It should include modules for:
	- Pipe IO
	- Array/Map manipulation
	- Advanced mathematical functions
	- date/time (using std::chrono)
	- file system management (file & directory read/write)
	- JSON parsing

	- D-Bus IPC
	- Networking over HTTP/HTTPS

	A lot of these are going to be excluded by default or are going to be developed way later down the line.
